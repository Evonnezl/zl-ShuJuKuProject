## 一、项目概述

### 1.1 项目背景

房产管理系统是面向高校/企事业单位房产管理科的信息化系统，用于替代传统的人工房产管理方式。系统围绕**房屋资源管理**和**用户住房申请管理**两大核心，支持**分房、调房、退房**三大业务流程，并提供**咨询与统计**辅助功能。

### 1.2 核心功能

| 功能模块 | 描述 |
| --- | --- |
| **分房申请** | 申请者提交申请 → 系统计算分数 → 分数达标则进入分房队列 → 按月统一分房 |
| **退房申请** | 删除住房记录 → 释放房屋 → 标记为空房 |
| **调房申请** | 根据条件确定住房等级 → 查找空房 → 释放原房 → 重新分配 |
| **咨询统计** | 查询分房阈值、房屋类型条件、单位面积租金；输出住房统计表；修改标准与租金 |

### 1.3 用户角色

| 角色 | 标识 | 权限 |
| --- | --- | --- |
| 房主 / 管理员 | `owner` | 管理房屋、审批申请、查看统计、修改标准 |
| 住户 | `tenant` | 提交分房/调房/退房申请、查看申请状态 |

## 五、后端设计

### 5.1 分层架构

```
Controller ──→ Service ──→ Mapper ──→ MyBatis XML ──→ MySQL
  (接口)       (业务)      (接口)        (SQL)         (数据)
```

### 5.2 Controller 层（REST API 端点）

| Controller | 路径前缀 | 主要端点 |
| --- | --- | --- |
| `UserController` | `/users` | GET 列表、CRUD |
| `HouseController` | `/houses` | GET 列表、CRUD |
| `ApplicationController` | `/applications` | GET 列表、审批分房/调房/退房 |
| `HousingRecordController` | `/housing-records` | GET 列表、CRUD |
| `HousingStandardController` | `/housing-standards` | GET 列表、CRUD |
| `StatsController` | `/stats` | GET 统计数据、阈值查询 |

### 5.3 Service 层（业务逻辑）

| Service | 核心职责 |
| --- | --- |
| `ApplicationService` | 申请审批流转：`approve()` 分房、`transfer()` 调房、`release()` 退房 |
| `HouseService` | 房屋 CRUD、状态管理 |
| `UserService` | 用户 CRUD |
| `HousingRecordService` | 住房记录管理 |
| `HousingStandardService` | 住房标准（面积-分数阈值）管理 |
| `StatsService` | 统计查询：房屋统计、申请统计、租金计算 |

### 5.4 Mapper 层

每个 Mapper 接口对应一个 MyBatis XML 映射文件，位于 `resources/mapper/` 目录下，定义具体的 SQL 语句。

### 5.5 Entity 实体类

| Entity | 对应表 | 关键字段 |
| --- | --- | --- |
| `User` | `user` | id, name, role, phone, email |
| `House` | `house` | id, title, status, area, rentPerM2 |
| `Application` | `application` | id, userId, type, status, score, createTime |
| `HousingRecord` | `housing_record` | id, userId, houseId, score, moveInDate |
| `HousingStandard` | `housing_standard` | id, area, minScore |

### 5.6 数据库连接配置

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/house_db
spring.datasource.username=root
spring.datasource.password=******
mybatis.mapper-locations=classpath:mapper/*.xml
```

## 八、核心业务流程

### 8.1 分房流程

```
用户提交分房申请
       │
       ▼
系统计算分数（部门 + 职称 + 家庭人数）
       │
       ▼
分数 ≥ 对应面积阈值？
   ├─ 否 → 申请拒绝
   └─ 是 → 进入分房队列（按分数排序）
              │
              ▼
         查询空房列表
              │
              ▼
         按优先级分配房屋
              │
              ▼
         创建 housing_record
         更新 house.status = '已分配'
         更新 application.status = 'APPROVED'
              │
              ▼
         输出分配结果，计算房租 = area × rent_per_m2
```

### 8.2 退房流程

```
用户提交退房申请
       │
       ▼
删除对应 housing_record
       │
       ▼
更新 house.status = '空房'
       │
       ▼
更新 application.status = 'APPROVED'
```

### 8.3 调房流程

```
用户提交调房申请（附带 original_house_id）
       │
       ▼
根据申请者条件确定住房等级
       │
       ▼
查找符合条件的新空房
       │
       ▼
释放原住房（原房 → 空房）
       │
       ▼
分配新房（新房 → 已分配）
       │
       ▼
更新 housing_record（user_id + new_house_id）
更新 application.status = 'APPROVED'
```

### 9.1 环境要求

| 组件 | 版本要求 |
| --- | --- |

### 9.2 启动步骤

**1. 初始化数据库**

```bash
mysql -u root -p < database/house_db.sql
```

**2. 启动后端**

```bash
cd backend/HouseBackend
./mvnw spring-boot:run
```

后端启动后监听 `http://localhost:8080`。

```bash
cd frontend
npm install
npm run dev
```
