# 房产管理系统

## 项目简介

本项目为数据库课程设计——房产管理系统。  

系统主要用于房屋资源管理和用户住房申请管理，支持分房、调房、退房和咨询统计功能。  
房产科将用户提交的申请表输入系统，系统会先检查申请表合法性，自动处理合法申请。

---

## 功能需求

系统核心功能：

1. **分房申请**
   
   - 根据申请者条件计算分数
   - 分数高于阈值则插入分房队列（按分数排序）
   - 每月最后一天分房：
     - 从空房文件读空房信息
     - 按分数高低将好房分配给符合条件的申请者
     - 更新住房文件、输出分配单、计算房租

2. **退房申请**
   
   - 删除住房文件和房租文件相关信息
   - 将房号写回空房文件

3. **调房申请**
   
   - 根据申请者情况确定住房等级
   - 查找符合等级的空房
   - 退掉原住房，并按分房逻辑分配新房

4. **咨询统计**
   
   - 查询阈值分数
   - 查询某类房屋条件
   - 查询房号的单位面积房租
   - 房产科打印住房统计表
   - 修改房屋居住条件和单位面积房租

## 数据要求（表结构）

| 表名    | 主要字段                             | 描述          |
| ----- | -------------------------------- | ----------- |
| 住房标准表 | 住房面积、最低住房分数                      | 按住房面积大小递增排列 |
| 房产表   | 房号、住房面积、分配标志、每平方米房租              | 按房号递增排列     |
| 住房表   | 户主、部门、职称、家庭人口、住房分数、房号、住房面积       | 已分配住房信息     |
| 住房要求表 | 户主 + [分房/调房/退房要求]                | 用户提交的申请     |
| 分房要求表 | 部门、职称、家庭人口、住房分数、要求住房面积           | 分房申请详细信息    |
| 调房要求表 | 部门、职称、家庭人口、住房分数、原住房面积、原房号、要求调房面积 | 调房申请详细信息    |
| 退房要求表 | 部门、房号                            | 退房申请详细信息    |
