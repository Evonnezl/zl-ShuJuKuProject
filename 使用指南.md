## 二、首次启动

### 步骤 1：初始化数据库

打开终端，执行项目中的建库脚本：

```bash
mysql -u root -p < database/house_db.sql
```

这会自动完成：删旧库 → 建新库 → 建表 → 建索引 → 插入测试数据。

> **注意**：如果之前已经建过库，此命令会先删除再重建，数据将被清空。如果只想增量升级已存在的库，请使用 `migration_v2.sql`。

### 步骤 2：启动后端

```bash
cd backend/HouseBackend
./mvnw spring-boot:run
```

看到 `Started HouseBackendApplication` 即启动成功，后端监听 **http://localhost:8080**。

### 步骤 3：启动前端

打开另一个终端：

```bash
cd frontend
npm install        # 仅首次执行
npm run dev
```

看到 `Local: http://localhost:5173/` 即启动成功。

### 步骤 4：打开浏览器

访问 **http://localhost:5173** 即可进入系统。

## 三、系统模块

### 3.1 房屋管理

- **路径**：点击顶部「房屋管理」按钮
- **内容**：查看所有房屋的房号、状态（空房/已分配）、面积、每平米租金、月租金
- **测试数据**：A101（50㎡空房）、A102（80㎡空房）、B201（120㎡已分配给张三）

### 3.2 申请管理

- **路径**：点击顶部「申请管理」按钮
- **内容**：查看所有分房/调房/退房申请，并执行审批操作
- **测试数据**：李四提交了一份分房申请（80㎡）
- **审批操作**：
  - **分房**按钮 → 计算分数 → 阈值校验 → 分配最优空房 → 生成住房记录 → 计算房租
  - **调房**按钮 → 释放原房 → 分配新房（仅 TRANSFER 类型申请可见）
  - **退房**按钮 → 删除住房记录 → 删除房租 → 房屋标记为空房（仅 RETURN 类型申请可见）

#### 分房流程详解

当你点击某个分房申请的「分房」按钮时，系统会自动完成以下步骤：

```
1. 计算分数
   公式：部门基础分 + 职称分 + 家庭人数 × 5
   
   - 部门：含"计算机/重点"=30分，含"学院/系"=20分，其他=10分
   - 职称：教授=50，副教授=40，讲师=30，助教=20，其他=10
   - 家庭：每多1人 +5分

2. 阈值校验
   根据申请面积查找对应的住房标准，分数 ≥ 最低分数才通过

3. 分配房屋
   从空房中取面积最大的（好房优先原则）

4. 创建住房记录
   写入户主、部门、职称、家庭人口、房号、住房面积、月租金

5. 写入房租表
   月租金 = 住房面积 × 每平方米月租金

6. 更新状态
   房屋 → "已分配"，申请 → "APPROVED"
```

> **点击前看一眼**：确保申请表里的部门、职称、家庭人数填写正确，这些值会直接影响分数计算。

### 3.3 用户管理

- **路径**：点击顶部「用户管理」按钮
- **内容**：查看用户列表（姓名、角色、电话、邮箱）
- **测试数据**：张三（管理员 owner）、李四（住户 tenant）

### 3.4 住房记录

- **路径**：点击顶部「住房记录」按钮
- **内容**：查看已分配的住房关系，包含户主、部门、职称、家庭人口、住房分数、房号、住房面积、月租金、入住日期

### 3.5 住房标准

- **路径**：点击顶部「住房标准」按钮
- **内容**：查看面积-分数阈值对照表
- **测试数据**：50㎡需≥60分、80㎡需≥80分、120㎡需≥100分

### 3.6 统计查询

- **路径**：点击顶部「统计查询」按钮
- **功能**：
  - **住房汇总**：空房数、已分配数、申请总数、总面积、平均租金
  - **按面积查阈值**：输入面积（如 80）可查询该面积对应的最低分数要求
  - **按房号查租金**：输入房号（如 A101）可查询该房间的每平米租金
  - **租金一览表**：列出所有房屋的月租金明细

## 四、测试业务流程

建议按以下顺序体验完整流程：

### 测试 1：分房

1. 打开「申请管理」页面
2. 找到李四的 `ALLOCATE` 申请（status = PENDING）
3. 点击「分房」按钮
4. 观察结果：
   - 申请状态变为 `APPROVED`
   - 「住房记录」中新增李四的住房信息
   - 「房屋管理」中对应房屋状态变为「已分配」

### 测试 2：调房

1. 先在数据库新增一条调房申请（或通过前端提交）
2. 申请类型为 `TRANSFER`，填写 `original_house_id`（原房号）
3. 点击「调房」按钮
4. 观察：原房释放为空房，新房分配成功

### 测试 3：退房

1. 提交退房申请（类型 `RETURN`，附带 `house_id`）
2. 点击「退房」按钮
3. 观察：住房记录被删除，房租记录被删除，房屋恢复为空房

### 测试 4：统计查询

1. 打开「统计查询」页面
2. 输入面积 50，点击查询 → 显示 50㎡ 需要 ≥60 分
3. 输入房号 A101，点击查询 → 显示每平米租金 10 元
4. 查看租金一览表 → 显示所有房屋的月租金

## 七、常见问题

**Q: 启动后端报 "Port 8080 already in use"**

A: 端口被占用，杀掉旧进程即可：
```bash
# Windows
netstat -ano | findstr :8080
taskkill /F /PID <PID号>
```

**Q: 前端页面加载不出数据**

A: 检查后端是否已启动（访问 http://localhost:8080/houses 看是否返回 JSON），以及后端是否开启了跨域（代码中已配置 `@CrossOrigin`）。

**Q: 点分房按钮没反应**

A: 查看后端控制台日志。常见原因：分数不够被拒绝（申请状态变为 REJECTED）、没有空房（申请保持 PENDING）。

**Q: 想修改评分公式**

A: 编辑 `backend/.../service/ApplicationService.java` 中的 `calculateScore()` 方法，重启后端即可生效。
